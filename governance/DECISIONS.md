# Governance Decisions

Use this file for governance-specific changes that affect how work is organized, recorded, reviewed, or handed off.

## Recording rules

- Record the date.
- State the decision plainly.
- State why it was made.
- State what files or workflows it changes.
- Link to supporting project decisions in `../DECISIONS.md` when relevant.

## Current governance baseline

- The repository is the project memory.
- Session continuity must survive context loss.
- Durable commits are more valuable than ephemeral chat context.
