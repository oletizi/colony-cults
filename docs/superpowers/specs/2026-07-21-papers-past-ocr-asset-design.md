# Design — Capture Papers Past source OCR as a first-class asset

Date: 2026-07-21
Branch: `feature/papers-past-ocr-asset`
Status: approved-in-conversation, pending written-spec review

## Problem

The Papers Past acquisition adapter (`bib acquire`, spec 015) mirrors only the
**page-image facsimiles** (page-master GIFs) to B2 + the archive with provenance.
It **parses** the article's OCR text (`parse.ts` `extractOcrText(#text-tab)`
returns `ocrText`) but then **discards** it — "OCR is out of scope as an acquired
asset" (operator decision, clarified 2026-07-19), on the rationale that OCR is
regenerated by the downstream OCR/translation pipeline.

That rationale is backwards for **newspapers**. Newspaper OCR is hard (dense
multi-column layout, small degraded type, broken baselines); NLNZ's Papers Past
provides **human-correctable** OCR that is far better than anything we would
regenerate from the GIFs. For a primary-source *text* corpus, the corrected
source OCR is the most valuable payload, and throwing it away to re-OCR later is
wasteful and lower-quality. The census (SRCH-0021…0032, 36 public-domain de Rays
articles queued for acquisition) surfaced this: we read that OCR to assess every
article, and the acquire adapter would drop it.

Today the OCR survives only incidentally, in the persisted governed-query
captures (`bibliography/repository-responses/papers-past-article/*.html`, the
`#text-tab` panel) referenced by each record's `metadataSnapshot.path` — not as
a structured, first-class corpus asset.

## Decision

Capture the source OCR text as a **first-class `AcquiredAsset`** in the same
acquire operation that mirrors the page-images, welding its metadata per
**Principle XV** (no orphan assets). Scope is **capture-only**; wiring the
downstream OCR/translation pipeline to *consume* the held OCR is a separable
follow-on (out of scope here).

### 1. Asset model (`src/model/acquired-asset.ts`)

- Add `'ocr-text'` to `ACQUIRED_ASSET_ROLES`. The `AcquiredAssetRole` union, the
  `isAcquiredAssetRole` runtime guard, and the fail-loud loader boundary
  (`@/bibliography/load-fields`) all derive from that tuple, so adding the member
  propagates automatically — no silent-accept path.

### 2. Adapter capture (`src/repository/papers-past/adapter.ts`)

The OCR is already in hand: `navigateAndParse` returns `parsed.ocrText`
in-memory at STEP 4/5 — **no extra fetch, no extra WAF exposure**. When
`ocrText` is present, add exactly one asset to the SAME verify-all-then-commit
flow (STEP 4 PHASE A verify / PHASE B commit) the page-masters use:

- `bytes` = UTF-8 encoding of `ocrText`
- `checksum` = sha256(bytes); `byteLength` = bytes.length
- `objectStoreKey` = `archive/papers-past/<id>/<sha256>.txt` (checksum-addressed
  → idempotent head-then-put; a re-run PUTs 0 duplicates)
- `mediaType` = `text/plain`
- `role` = `ocr-text`
- `sequence` = `0` (ahead of the page-master images, which are `1..N`)
- provenance sidecar written the same way as the page-masters
  (`archive/papers-past/<id>/<sha256>.yml`)

Because the OCR text is already parsed, its PHASE A is just the checksum (no
byte-fetch); PHASE B is the idempotent head-then-put. It joins the all-or-nothing
commit, so a mid-sequence failure still leaves **zero orphaned objects**, and the
SSOT `assets[]` entry welds in the same operation (Principle XV).

### 3. Fidelity & edge cases

- **Fidelity:** store the `#text-tab` text **faithfully** as `extractOcrText`
  returns it. Do NOT whitespace-collapse (that was display-only formatting in the
  census scripts). Preserve the source text for the corpus.
- **Absent OCR** (`ocrText === undefined` — e.g. an image-only article): **not
  fatal**. Acquire the page-masters and emit no `ocr-text` asset. The absence is
  observable (no `ocr-text` asset in the result) and logged — newspaper articles
  should always have OCR, so absence must not block mirroring the facsimile but
  should be visible. (Reuse the existing result/logging surface; do NOT add a new
  `AcquisitionResult` field for this unless one already exists.)
- **Rights:** covered by the record's existing fail-closed `public-domain` gate
  (STEP 1). The same assessment that permits mirroring the images permits the
  OCR text; no separate rights handling.
- **Dry-run:** unchanged — returns empty `assets` (no OCR put, no record
  mutation), same as today.

### 4. Tests (extend the hermetic spec-015 suite — injected fakes, 0 network)

- Scenario 1 (public-domain acquire) gains an assertion: acquire puts **N
  page-master GIFs + 1 `ocr-text` .txt** whose bytes are the parsed OCR at the
  deterministic key; the fake `ObjectStore` records the extra PUT.
- Idempotency: a re-run with the OCR object already present at the matching
  checksum → **0 duplicate** OCR puts.
- Dry-run: **0** OCR puts, 0 record mutation.
- OCR-absent: fake article HTML without a `#text-tab` panel → page-masters
  acquired, **no** `ocr-text` asset, **no throw** (absence observable in the
  returned assets).
- Loader round-trip: an `ocr-text` asset serialized into a source record loads
  back through `@/bibliography/load-fields` (the new role is accepted, not
  rejected as unknown).

### 5. Governance

A focused, hermetically-testable adapter + model change → **direct TDD on
`feature/papers-past-ocr-asset`** (not a full Spec Kit spec). This design doc is
the durable record. It closes the "source OCR discarded" gap the census
surfaced, and makes the pending acquisition of the 36 de Rays articles capture
the corrected NLNZ OCR instead of throwing it away.

## Out of scope

- Wiring the downstream OCR/translation pipeline to consume the held `ocr-text`
  asset (separable follow-on).
- Any other adapter (Gallica / museum / Internet Archive) — monograph re-OCR is
  acceptable; this is the newspaper case.
- The live acquisition itself, which separately requires the operator's
  `COLONY_ARCHIVE_ROOT` (per-session archive clone) + B2 credentials.

## Acceptance

- Acquiring a public-domain Papers Past article with OCR present yields, in ONE
  operation, N `page-master` GIF assets **plus** one `ocr-text` `text/plain`
  asset, all in B2 + the archive with provenance and a complete SSOT record
  (no orphan, Principle XV).
- The `ocr-text` bytes are the faithful `#text-tab` OCR; the object key is
  `archive/papers-past/<id>/<sha256>.txt`; re-runs are idempotent (0 duplicate
  puts).
- An OCR-absent article acquires page-masters only, with no throw and a note.
- `npm test` + `npm run typecheck` green; the hermetic suite makes 0 network
  calls and never mutates the real object store.
