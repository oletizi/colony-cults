# Governance

This repository is the project memory.

The working rule is simple: optimize for durable commits, not chat responses.

## Core principles

- Commit early.
- Commit often.
- Push frequently.
- Evidence before narrative.
- Preserve provenance.
- Respect copyright.
- Never lose work.
- Never make the same discovery twice.
- Leave the project better than you found it.
- The project must survive context loss.

## Operating implications

- Uncommitted work is temporary.
- Committed work is durable.
- Pushed work is recoverable.
- Every factual claim should move toward a source ID.
- Notes should separate evidence, interpretation, and uncertainty.
- Copyright uncertainty blocks mirroring, not cataloging.
- Session state and all durable knowledge should be left in versioned repo files — NEVER in a private per-machine agent-memory store (Constitution XIII). This repository is the project memory; a private, unshared store throws hard-won knowledge away.

## Minimum session standard

At the start of a session, read the project dashboard and current logs before changing direction.

At the end of a session:

- update the research log
- record any durable decisions
- refresh current state and next actions
- commit the work
- push the work
