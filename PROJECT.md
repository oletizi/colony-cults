# Project Dashboard

## Mission

Build a durable, source-driven research archive for nineteenth-century utopian, speculative, and fraudulent colonization schemes, beginning with the Marquis de Rays and the Free Colony of Port Breton.

## Repositories

- `oletizi/colony-cults`: public research, bibliography, notes, schemas, source metadata, and project management.
- `oletizi/colony-cults-archive`: private preservation mirror for legally acquirable digital assets.

## Current phase

**Phase 1: Foundation and Source Acquisition**

The current priority is to establish research infrastructure and acquire/catalog legally mirrorable primary sources for the Port Breton case.

## Active milestone

**Milestone 2: Port Breton Public-Domain Acquisition**

Initial targets:

1. *La Nouvelle France* recruitment newspaper.
2. Paul de Groote, *Nouvelle-France: Colonie libre de Port-Breton, Océanie*.
3. A. Baudouin, *L'aventure de Port-Breton et la colonie libre dite Nouvelle-France*.
4. French trial/legal pamphlets.
5. Australian newspaper coverage via Trove.
6. Nineteenth-century maps of New Ireland and surrounding region.

## Completed milestone

**Milestone 1: Project Infrastructure**

Completion criteria met:

- Public repository contains project dashboard, roadmap, research log, decision log, and agent/contributor guidance.
- Private archive repository contains preservation policy, acquisition register, and metadata stubs for initial primary sources.
- Source acquisition workflow is documented.

## Working rules

- The public repository is the source of truth for project state.
- The private archive repository stores only assets that may be legally mirrored.
- Every factual claim should eventually trace to a source ID.
- Every mirrored asset should have provenance metadata and a checksum.
- Copyright uncertainty should block mirroring, not cataloging.

## Session protocol

At the start of a work session:

1. Read `PROJECT.md`, `ROADMAP.md`, `DECISIONS.md`, and `RESEARCH_LOG.md`.
2. Identify the active milestone and highest-priority unfinished task.
3. Make small, atomic commits.
4. Update `RESEARCH_LOG.md` before stopping.
5. Update `PROJECT.md` or `ROADMAP.md` if priorities change.
